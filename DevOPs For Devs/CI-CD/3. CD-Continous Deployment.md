# CD - Continous Delivery

Automating all the deployment build steps.

Steps include:

1. Checkout the code locally

2. Build a new Docker Image

3. Update k8s manifest file

4. connect to k8s cluster

5. Applt deployment




## Automating End 2 End Testing

To ensure that new merge/release doesnt break any other services.

To ensure project functions correclty in real-world applications.

 

We automate E2E in order in to minizmes the error cause by human intervention.



The code is tested at every stage (DEV, Staging ,Production). this process is kown as Continuos Delievery (`CD`)





> After `CI/CD` Devlopers feel confident of the live release sinxe the app has no errors (Thanks to `CI` )  and all the functionalities works as expected (Thanks to `CD` )
