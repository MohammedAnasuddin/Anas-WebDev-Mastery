# CI-Continuos Integration




## Fixing Test Failures before Merge

It would be great if the branches are tested before merging so that test won't fail after merging.

> If multiple branches are merged and tests fail it would be diffcult to identify which branch caused the failure.
> 
> Hence testing independently will ensure branch doesnt fail after merging. 




### A Better Approach

Running these tests on every commit of branch will let us know which code failed the tests.  



> `TIP`
> 
> It is encouraged to commit frequently and test so that the broken code identified quickly.  



Formula : `Commit small Frequently -> Test Fast -> Fix Early`

This is known as Continuos Integration (`CI`).  So, it is Keeping changes fast and Integrate regularly.

 

## Traditional Flow v/s CI

| **Traditional**                             | **CI**                          |
| ------------------------------------------- | ------------------------------- |
| Complex, entangled issues                   | Issues caught early & often     |
| Bugs found late by testers                  | Bugs are small and isolated     |
| Delayed feedback → stress before release    | Devs fix things immediately     |
| Merge conflicts → devs overwrite each other | Less stress near release        |
| Frustrated developers                       | Clean merges → happy developers |
