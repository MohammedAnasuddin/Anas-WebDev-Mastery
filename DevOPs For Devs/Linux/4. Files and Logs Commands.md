# Handling Files and Logs In Linux

## Logs in Linux

A **log** is a **written(text) record of events**. A log is a message written by a program to remember what happened, when it happened, and how it happened.

Log files are **plain text files** that store records of events generated by:

- Operating system

- Servers

- Applications

- Databases

- Background services

They help engineers **observe, debug, and audit** system behavior.

`.log` is a **convention**, not a rule.

- It indicates the file contains log data

- **Linux does not require this extension**

- Programs choose it for clarity.

A log entry usually has:

- Timestamp

- Severity level (INFO, WARN, ERROR)

- Message

## `cat` : Concatenate files content and print on the standard output

use `-` to print user input.

## `more`:  more  is a filter for paging through text one screenful at a time.

used to read large text with scroll like feature.

can view multiple files at a time provides a good separation.

## `less` : opposite of more

similar to more (1), but which  allows  backward movement in the file as well as forward movement.

less does not have to read the entire input file before  starting, so with large  input  files it starts up faster than text editors.

> `NOTE`
> 
> To exit form `more` or `less` hit `q`

### Searching in `less`

Search is performed in 3 ways usng regular expressions

1. `/pattern` : Search forward in the file form the current position
   
   - Press `n` → next match forward
   
   - Press `N` → previous match

2. `?pattern`: Search  backward in the file from the current position.

3. `&pattern`: Display only lines which match the pattern

> `NOTE`
> 
> The **current position** is:
> 
> **The line at the top of your screen (your current view) + an internal pointer inside the file**

## `tail`  Print  the  last  10  lines of each FILE to standard output.

1. `tail` jumps to the **end of the file**

2. It reads **backwards in chunks**

3. It counts newline characters (`\n`)

4. Once it has found **N lines**, it stops

5. It prints only those lines

> `NOTE`
> 
> To modify no. of lines in output use `-n <number>` 
> 
> > `TIP`
> > 
> > To get all lines from a specific line use `-n +<starting_line>`

### Observing Real-time file updates using `tail -f`

`tail -f` prints appended content as file gets edited, but if a file is truncated, it resets and prints the whole content of the file.

## `grep`:   searches  for  PATTERNS  in  each  FILE.

patterns separated by newline characters,  and  grep  prints  each line that matches a pattern.

> `NOTE`
> 
>  If no FILE is given, recursive searches examine the working directory, and nonrecursive searches read standard input.

### Important Flags

#### Getting Patterns

- `-F` : Interpret  PATTERNS  as  fixed  strings,  not  regular expressions.

- `-f` : Obtain patterns from FILE, one per line.

#### Matching Conditions

- `-i`:  Ignore  case  distinctions

- `-v`: Invert the sense of matching, to select non-matching lines.

- `--exclude-from=FILE`: Skip files whose base name matches any of the file-name.

- `-r`: Read all files under each directory recursively.

#### Formatting the print

- `-c`: instead print a count  of  matching lines  for  each  input  file.

- `-L`: Print file names of whoose content *does not* match the pattern.

- `-l`: Print file names of whoose content matches the pattern.

- `-o`: Print only matched content.

- `-H`: Prints the file name for each match

- `-n`: Prints the line number of the match.
