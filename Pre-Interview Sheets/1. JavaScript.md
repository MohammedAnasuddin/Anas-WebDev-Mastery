



# ✅ **JAVASCRIPT INTERVIEW QUICK-REVISION NOTES**

*(Foundations → Traps → One-liners)*

---

## 1️⃣ What is JavaScript

**Best answer:**

> JavaScript is a high-level, interpreted programming language that runs in environments like browsers and Node.js to control behavior and interaction in applications.

**Avoid:**

- “Used to make websites interactive”

- Starting with “isomorphic”

---

## 2️⃣ Compiled vs Interpreted

**Safe answer:**

> JavaScript is conceptually an interpreted language, but modern engines use JIT compilation to improve performance.

**Do NOT start with engine internals.**

---

## 3️⃣ Execution Context

**Definition:**

> An execution context is the environment in which JavaScript code is evaluated and executed.

### Types:

1. Global Execution Context

2. Function Execution Context

3. Eval Execution Context (rare)

---

## 4️⃣ Phases of Execution Context

### Two phases:

1️⃣ **Creation Phase**

- Memory allocated for variables & functions

- `this` is set

- Variables are `undefined`

2️⃣ **Execution Phase**

- Code runs line by line

- Variables get actual values

---

## 5️⃣ Hoisting

**Best definition:**

> Hoisting is JavaScript’s behavior where declarations are processed during the creation phase before code execution.

### Key rules:

- `var` → hoisted + initialized with `undefined`

- `let` / `const` → hoisted but **not initialized** (TDZ)

- Function declarations → fully hoisted

- Function expressions → variable hoisting only

---

## 6️⃣ Hoisting Traps

```js
console.log(a); // undefined
var a = 10;
```

```js
console.log(b); // ReferenceError
let b = 10;
```

```js
foo(); // works
function foo(){}
```

```js
bar(); // TypeError
var bar = function(){}
```

## 7️⃣ Scope

**Definition:**

> Scope determines where variables and functions are accessible.

### Types:

- Global scope

- Function scope

- Block scope

❌ No “object scope”

---

## 8️⃣ Scope Chain

> If a variable is not found in the current scope, JavaScript looks up through parent scopes until the global scope.

**Important:**

> Scope chain is created at **function definition time**, not execution time.

---

## 9️⃣ Closures

**Perfect definition:**

> A closure is created when a function remembers and continues to access variables from its lexical scope even after the outer function has finished execution.

**Golden rule:**

> Closures persist only if a reference to the inner function is retained.

---

## 🔟 `this` Keyword

**Definition:**

> `this` refers to the object that is currently executing the function.

**Key rule:**

> `this` is determined by how a function is called, not where it is defined.

### Normal function:

- Method call → object before dot

- Plain call → global / undefined (strict)

### Arrow function:

> Arrow functions capture `this` from their surrounding lexical scope.

---

## 1️⃣1️⃣ Sync vs Async

- **Synchronous** → blocks execution

- **Asynchronous** → does not block, continues execution

**Why async exists:**

> To prevent blocking the main thread and freezing the UI.

---

## 1️⃣2️⃣ Event Loop

**Definition:**

> The event loop is a runtime mechanism that moves tasks from queues to the call stack when it is empty.

### Execution order:

1. Call stack (sync)

2. Microtasks (Promises)

3. Macrotasks (setTimeout)

---

## 1️⃣3️⃣ Async Traps

```js
console.log("A");
setTimeout(() => console.log("B"), 0);
Promise.resolve().then(() => console.log("C"));
console.log("D");
```

**Output:**

```
A
D
C
B
```

---

## 1️⃣4️⃣ Pass by Value vs Reference

**Correct statement:**

> JavaScript always passes arguments by value.  
> For objects and arrays, the value passed is a reference.

**Meaning:**

- You can mutate contents

- You cannot reassign the original reference

---

## 1️⃣5️⃣ Arrays

> Arrays are passed by value, but that value is a reference.

---

## 1️⃣6️⃣ Shallow vs Deep Copy

### Shallow Copy:

- Copies top-level

- Nested objects are shared

### Deep Copy:

- Fully independent copy

- No shared references

**One-liner:**

> Shallow copy shares nested references, deep copy does not.
