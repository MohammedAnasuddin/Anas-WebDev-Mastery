

# ✅ **NODE.JS & EXPRESS — INTERVIEW QUICK NOTES**

---

## 1️⃣ What is Node.js

**Best answer:**

> Node.js is a JavaScript runtime environment that allows JavaScript to run outside the browser, built on Chrome’s V8 engine.

**Key idea:**

- JS on server

- Not a framework

- Not JavaScript itself

---

## 2️⃣ Why Node.js was created

**Answer:**

> To enable JavaScript for server-side development and handle many concurrent connections efficiently.

**Core reason:**

- Non-blocking I/O

- Event-driven architecture

- I/O-heavy applications

---

## 3️⃣ Non-Blocking I/O

**Definition:**

> Non-blocking I/O allows Node.js to continue executing code without waiting for I/O operations to finish.

**Mental model:**

> Start I/O → don’t wait → handle result later

---

## 4️⃣ Single-Threaded Nature

**Fact:**

- JavaScript runs on a single main thread

**Still handles concurrency because:**

- I/O is offloaded to runtime

- Event loop schedules callbacks

❌ Not parallel JS execution

---

## 5️⃣ libuv

**Role:**

- Handles async I/O

- Uses OS async APIs

- Maintains internal thread pool (for fs, crypto, etc.)

**Important:**

> JavaScript never runs in libuv threads.

---

## 6️⃣ Event Loop

**Definition:**

> The event loop continuously checks the call stack and callback queues and pushes callbacks to the stack when it is empty.

**Purpose:**

- Enables async, non-blocking behavior

---

## 7️⃣ Blocking vs Non-Blocking Code

| Blocking            | Non-Blocking          |
| ------------------- | --------------------- |
| Waits for operation | Continues execution   |
| Freezes event loop  | Keeps loop responsive |

---

## 8️⃣ Async Handling in Node.js

**Ways:**

- Callbacks (legacy)

- Promises

- Async / Await (preferred)

**Rule:**

> Async/await is syntax sugar over promises.

---

## 9️⃣ Error Handling (Node.js)

- **Callbacks** → error-first pattern

- **Promises** → `.catch()`

- **Async/Await** → `try...catch`

**Formula:**

> async code → catch error → handle gracefully

---

## 🔟 Modules in Node.js

**Definition:**

> A module is a reusable, isolated piece of code that encapsulates functionality.

**Key concepts:**

- Each file is a module

- Private scope by default

- Explicit exports

---

## 1️⃣1️⃣ CommonJS vs ES Modules

**CommonJS**

- `require`, `module.exports`

- Runtime loading

- Default in older Node

**ES Modules**

- `import`, `export`

- Static analysis

- Modern standard

- Needs `"type": "module"` or `.mjs`

**Memory rule:**

> CommonJS = runtime  
> ES Modules = static

---

## 1️⃣2️⃣ What is Express.js

**Definition:**

> Express.js is a minimal and flexible web framework built on top of Node.js.

**Why used:**

- Routing

- Middleware

- API development

- Request/response handling

---

## 1️⃣3️⃣ Middleware

**Definition:**

> Middleware is a function that runs during the request–response cycle and can modify the request, response, or pass control forward.

**Uses:**

- Auth

- Logging

- Validation

- Parsing JSON

---

## 1️⃣4️⃣ `next()`

**Purpose:**

> Passes control to the next middleware or route handler.

**Rule:**

> No `next()` → request stops

---

## 1️⃣5️⃣ Error Handling in Express

**Correct approach:**

- Centralized error-handling middleware

**Pattern:**

> error → `next(err)` → error middleware

**Signature:**

```js
(err, req, res, next)
```

---

## 1️⃣6️⃣ REST API

**Definition:**

> REST is an architectural style for designing APIs using resource-based URLs and standard HTTP methods.

**Mental model:**

> REST = resource-based design

---

## 1️⃣7️⃣ REST Principles (Core)

- Stateless

- Client–Server separation

- Resource-based URLs

- Uniform interface

**Stateless means:**

> Server stores no client context between requests

---

## 1️⃣8️⃣ HTTP Methods

| Method | Purpose          |
| ------ | ---------------- |
| GET    | Read data        |
| POST   | Create data      |
| PUT    | Replace resource |
| PATCH  | Partial update   |
| DELETE | Remove resource  |

---

## 🔑 **NODE + EXPRESS INTERVIEW FORMULAS (MEMORIZE)**

- JS single-threaded + async I/O = concurrency

- Non-blocking ≠ parallel JS

- Event loop = scheduler

- Module = private scope + explicit export

- Express = Node + structure

- Middleware = request pipeline

- `next()` = continue chain

- REST = stateless resource-based API

---

## 🧠 **INTERVIEW WARNINGS (VERY IMPORTANT)**

❌ Node is not multi-threaded JS  
❌ Express is not required for Node  
❌ REST is not JSON-only  
❌ try/catch alone is not Express error handling
